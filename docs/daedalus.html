Benchmarks — M4 MacBook Air 16GB, Qwen3.5-9B 4-bit

The middle path is the fast path.

Four consecutive 20k-token prefills, cooled to nominal before each arm. Unpaced, the Air throttles inside the first prompt and keeps degrading. Paced, the silicon never throttles — the same work costs 20% fewer GPU-seconds.

Stock engine (governor off)647s GPU · rate decays 133 → 119 tok/s

50% of the run throttled at HEAVY thermal pressure. Every round slower than the last.

daedalus (governor on)515s GPU · stable 157 tok/s

21% throttled. Recovers to NOMINAL between rounds. Rate never decays.

Every number measured on real hardware — raw traces in bench/results/, reproduce with bench/thermal_validation.py. With the prefix cache, a given prompt pays its cold prefill exactly once, ever.

Why it exists

Built for the machine everyone actually owns.

Every other MLX engine is benchmarked on fan-cooled Studios and Maxes. daedalus treats the fanless Air's thermal envelope as the design constraint.

▲▲

Persistent prefix cache

Your agent's system prompt is prefilled once, ever — saved to disk as KV state, recalled on every later turn, across restarts. Correct even for hybrid-attention models (Qwen3.5, Gemma 4) where stock servers silently miss.

⟲

Resumable prefill

Checkpoints every 4k tokens. A timeout, crash, or a client's hard 300-second deadline never repeats completed work — the retry resumes exactly where it stopped.

◐

Thermal governor

Reads macOS's five-level thermal-pressure signal live — no sudo — and duty-cycles prefill against it. Three profiles: performance, balanced, cool. The first of its kind in local inference.

≋

Keepalives with telemetry

SSE heartbeats stream progress, thermal state, and ETA from the first second of a long prefill, so idle-timeout clients never disconnect.

ƒ

Strict tool-call streaming

OpenAI-format deltas with explicit indexes, never an empty tool_calls array — the malformed shapes that hang agents are structurally impossible.

◔

Reasoning separation

Chain-of-thought from thinking models is split into reasoning_content — it never leaks into the visible reply. pi, OpenCode, and any OpenAI-compatible client just work.

Install

Up in three commands.

$uv tool install --editable . --with "transformers>=5.0,<5.11"
$daedalus serve --model mlx-community/Qwen3.5-9B-MLX-4bit --port 8484

Then point pi, OpenCode, or any OpenAI-compatible agent at http://127.0.0.1:8484/v1
