The 3,000-year-old design doc

Three ways to cross the Aegean

Daedalus gave his son one engineering constraint: fly the middle course. Every local inference engine today ignores it.

Fly too high

Full-power prefill, no pacing. Your Air hits HEAVY thermal pressure inside the first 20k-token prompt, the clocks throttle, and throughput decays every round after.

Fly too low

Throttle everything and the client drowns waiting — OpenCode kills any request at 300 seconds, streamed or not. Caution without a cache is just a slower failure.

Race while cool. Ease off as live thermal pressure rises. Never re-fly a league already flown. Cool silicon runs faster when it runs — the middle path is the fast path.

The craft

Four inventions, one workshop

the labyrinth

A prefix cache that survives everything

Your agent's giant system prompt is prefilled once, ever — persisted to disk as KV state and recalled exactly on every later turn, across crashes and reboots. Correct even for hybrid-attention models (Qwen3.5, Gemma 4) where stock servers silently miss every time.

the thread

Resumable prefill

Checkpoints every 4k tokens mean a timeout, a crash, or OpenCode's hard 300-second deadline never restarts the maze — the retry picks up the thread exactly where it dropped.

the middle path

The first thermal governor in local inference

Reads macOS's five-level thermal-pressure signal live (no sudo) — including the two levels Apple's public API hides — and duty-cycles prefill against it. Three tempers: --profile performance | balanced | cool.

the herald

Keepalives and tool calls that never hang

SSE heartbeats with progress, thermal state, and ETA from the first second of prefill. Tool calls stream as strict OpenAI deltas — indexed, never empty — so pi, OpenCode, and any OpenAI-compatible agent just work.
